# Dead Docs Policy

Politica para documentacion obsoleta, duplicada o sin uso claro.

## Opciones permitidas

- Archivar: mover a una ubicacion historica solo con justificacion y enlaces de reemplazo.
- Enlazar: mantener la doc como indice si todavia orienta bien.
- Fusionar: trasladar contenido vigente a la fuente normativa y dejar nota de reemplazo.
- Borrar: solo con justificacion humana explicita y diff revisable.

## Reglas

- No borrar documentacion por parecer antigua sin comprobar enlaces y uso.
- No dejar dos specs normativas contradictorias para el mismo flujo.
- Si una regla sigue activa, debe vivir en una fuente normativa o ADR.
- Si una regla es historica, debe indicarlo claramente.
- Si una doc solo repite harness, preferir enlace al harness.

## Confirmacion humana

Pedir aprobacion antes de borrar, mover masivamente o cambiar la autoridad normativa de una doc.
