# Onboarding IA del proyecto

## Que leer primero

1. [AGENTS.md](../AGENTS.md)
2. [docs/ia_workflow.md](ia_workflow.md)
3. [docs/documentation_governance.md](documentation_governance.md)
4. Documento tematico afectado.
5. ADRs relacionados en [docs/adr/](adr/).

## Que nunca romper

- Canon actual.
- Reglas invariantes.
- Anti-patrones.
- Decisiones activas.
- Sincronizacion `AGENTS.md` / `agents.md`.

## Como proponer un cambio

- Identificar dominio.
- Revisar matriz de impacto.
- Revisar ADRs.
- Marcar estado.
- Actualizar changelog si cambia una decision.
- Validar documentacion.

## Errores comunes de IA

- Crear APIs paralelas.
- Duplicar navegacion.
- Ignorar mobile first.
- Inventar soporte no documentado.
- Modificar comportamiento sin actualizar documentacion.
- Crear reglas sin Decision ID.
